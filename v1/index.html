Nemojs

selenium-webdriver automation for node.js

Introducing Nemo

Nemo can be used as a stand-alone tool for convenience scripts, or paired with test runner/task runner modules to create a UI test automation suite. Nemo itself has very few lines of code. But it provides an excellent framework for you to build from, whatever your automation needs.

Plugins

Nemo uses a plugin based architecture to allow for a lightweight core module, customization, and the opportunity for YOU to contribute easily to the Nemo ecosystem. The essential (in our opinion) plugin to put the diving helmet on Nemo is nemo-view. You can read about it in its README. You can also see it in action in the sample application. You can also easily add it to your own node.js application using the generator-nemo yeoman generator.

Configurability

Using the clever, concise and powerful confit configuration module and its shortstop handler syntax, you have many elegant ways to configure - and override - Nemo settings.

Quick Start

  • Create an empty npm project (press enter through all prompts): 
    $ npm init
  • Install Nemo: 
    $ npm install --save nemo
  • Copy the following code into index.js 
    var Nemo = require('nemo');
var nemo = Nemo({
  'driver': {
    'browser': 'chrome'
  }
}, function (err) {
  //always check for errors!
  if (err) {
    console.log('Error during Nemo setup', err);
    return false;
  }
  nemo.driver.get('https://www.paypal.com');
  nemo.driver.getCapabilities().
    then(function (caps) {
      console.info("Nemo successfully launched", caps.caps_.browserName);
    });
  nemo.driver.quit();
});
  • Run it: 
    $ node index.js
Nemo successfully launched chrome
  • Now, use a shorstop handler to allow configuring the browser from argv by changing the following line: 
    'browser': 'chrome'
    to 
    'browser': 'argv:browser'
  • Run it again: 
    $ node index.js --browser firefox
Nemo successfully launched firefox

This is barely getting your toe in the water so far. But it feels good right?
